# 交接文档（第二十批·快修）：401 引导 + 解析选项布局（2026-07-08）

> 用户上线后实测反馈的两个体验问题。小批快修，**每项 `npm test` 全绿后单独 commit**（gpg 用 `-c commit.gpgsign=false`）。**绝对不要 git push**（我来合并上线）。

## T1：API 401 时给出明确的 Token 引导（高优）

**现状**：P0 鉴权上线后，浏览器没填 Token 时所有 API 401，前端把菜谱列表静默显示成"还没有菜谱"空状态——用户以为数据丢了（真实事故，用户被吓到）。

**要做**：
1. 前端统一处理 401：任何 API 请求收到 401 时，页面顶部出现**显眼的常驻横幅**：「🔒 服务器已开启鉴权，请在设置中填写 API Token」+ 一个「去设置」按钮（点击直达设置页并高亮 Token 输入框）；横幅在 Token 保存成功后消失。
2. 菜谱列表在 401 时**不要渲染"还没有菜谱"空状态**，改为显示同款引导提示（区分"真的空"和"没权限"两个状态）。
3. 填入 Token 保存后自动重拉列表（不需要手动刷新）。
4. i18n（zh+en）；纯逻辑（401 状态判定/横幅显隐）配前端测试；现有测试全绿。

## T2：解析选项行布局修复

**现状**：首页解析卡片里「读画面字幕」「提取画面截图」两行，checkbox 和右侧说明文字之间隔了巨大空白（checkbox 居中偏左、文字靠右），观感破碎（见用户截图）。

**要做**：checkbox 与标签、说明文字**左对齐排成一行**（label 包裹 checkbox 保证可点击区域），两行样式一致；移动端换行自然；暗色模式正常；不改功能语义。若这是 M1 样式改造的回归，顺带检查同类 checkbox 行有没有别处同样问题。

## T3：README 双语补「访问模型」说明（面向开源使用者与实例主人）

在 README「多用户」或「快速开始」附近补一小节，讲清三层访问模型（内容要点）：
1. **想用庖丁的人**：自托管自己的实例（Docker / 本地方案），token 是每个实例自己生成的私有凭证，**不在代码仓库里**；
2. **实例主人给家人朋友开权限**：用 `PAODING_API_TOKENS="ming:tokenA,hua:tokenB"` 按人发放命名 token——每人独立的收藏/购物清单/计划（菜谱库共享），删掉某人的 token 重启即撤销；
3. **不建议把私有实例开给陌生人**：本地模型算力有限（并发上限+排队+限流是防滥用兜底，不是公开服务能力）；公开分享用只读分享页 `/r/` 或让对方自托管。
同步检查 `.env.example` 对 PAODING_API_TOKENS 的注释是否讲清了这个场景。
