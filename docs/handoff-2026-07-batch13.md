## M1：UI 样式系统打磨（用户点名；对标 Deglaze/Mela 的精致度）

纯 CSS + 必要的类名微调，**不引前端框架**，不破坏现有前端测试：

1. **Design tokens 统一**：styles.css 顶部整理 CSS 变量——间距梯度（4/8/12/16/20/24）、圆角梯度、阴影层级（卡片/弹层/浮层）、字号梯度；把散落的魔法数字归拢到变量（渐进式，改到哪归到哪，不要求一次穷尽）。
2. **交互态完善**：所有可点元素补 hover / active / focus-visible 态（键盘可达性）；触控目标 ≥44px（移动端主按钮）。
3. **过渡动效**：卡片/弹窗/tab 切换加 150–250ms ease 过渡；尊重 `prefers-reduced-motion`。
4. **视觉层次**：菜谱卡片（图/标题/标签/评分的层次）、双板块（批量/单份）板块头、工具卡"无替代"标记、营养卡——统一卡片语言。
5. **空状态与加载**：菜谱库为空、搜索无结果、技法页为空时给友好空状态（文案+引导动作）；解析中的任务卡加骨架/脉冲效果。
6. **暗色模式一致性**：逐组件核对暗色变量，特别是新加的（单位气泡、双板块、导入提示、打印按钮）。
7. 打印样式不受影响（回归 G2 的 @media print）。

## M2：菜谱集 Cookbooks（ReciMe/Paprika 标配）

1. 用户可创建菜谱集（如 早餐/宴客/减脂），一道菜可属多个集；数据存 userdata（per-user，走 rev 同步）。
2. 首页新增「菜谱集」入口：集列表（封面取集内第一道菜的图，无图用色块+首字）→ 集内菜谱列表；菜谱详情页可勾选加入/移出集。
3. 集的增删改名；删除集不删菜谱。
4. 纯逻辑（集合增删/成员管理）配前端测试；i18n。

## M3：食材库存 Pantry v1（Paprika/SuperCook 标配）

1. 「我的食材」页：添加/删除库存食材（名称+可选数量备注），常见食材快捷 chips（葱姜蒜/生抽/老抽/料酒/糖/盐…一键添加）；存 userdata per-user。
2. **「能做什么」**：本地匹配——对每道菜计算食材覆盖率（库存命中/总食材数，忽略"适量"类调料可配置），按覆盖率排序展示，缺的食材标出来（"还差：豆瓣酱、五花肉"）；纯函数配测试。
3. 可选 AI 推荐按钮：把库存清单发给 LLM 出 3 个菜思路（走鉴权+限流，标注 AI 生成）。
4. 购物清单联动：加入购物清单时已在库存的条目自动标记"已有"。

## M4：AI 菜谱变体（Samsung Food 的杀手级功能）

1. 菜谱详情页「变体」按钮：素食化 / 更健康（少油少糖）/ 低盐 / 替换指定食材，LLM 基于原菜谱生成修改版。
2. 变体落库为**新菜谱**：标 `variant_of: 原菜谱id` 与变体类型，标题自动加后缀（如「宫保鸡丁（素食版）」）；详情页显著标注「AI 改编，未经实做验证」+ 回链原菜谱；原菜谱页列出已有变体。
3. 走鉴权+限流；LLM 输出复用 structureRecipe 的 schema 校验路径（含 tools/营养清洗）；诚实原则——变体的 why 讲解允许 LLM 重写但保持标注。
4. LLM stub 测试：变体落库、variant_of 链接、清洗生效。
