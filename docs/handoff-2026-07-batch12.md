## 环境须知

- CLI 入口：`node bin/paoding.mjs "<URL或本地文件>" --images`（--images 开画面配图；读画面字幕开关看 CLI help / .env）
- 产出落在 `recipes/`（worktree 内，隔离于生产）
- 视频用 yt-dlp 搜索语法自选：`ytsearch5:关键词`（YouTube）；B站直接贴 URL，若 412 加 `.env` 的 `PAODING_COOKIES_FROM_BROWSER=chrome`
- **选短视频**（1–8 分钟最佳），本地模型跑一条约几分钟；一次只跑一条，别并行
- 网络失败/平台反爬拿不到视频时换平台换视频，不要死磕同一条

## L2：特性验证矩阵（核心）

每条视频跑完后**检查产出 JSON 与 Markdown**，对照验收点记录 ✅/❌/⚠️：

| 场景（自己搜视频） | 验收点 |
|---|---|
| 饮品/奶茶类，最好片尾有配料表卡 | K1：视觉转录含【画面配方卡】且用量进菜谱并留痕；K2：phase 双板块/batch_info 是否合理触发（或正确不触发） |
| 含香料的炖菜/卤味（口播含 八角/桂皮/白芷 等） | J1：食材名无同音错字，纠错有 note 留痕；J2：拇指长/指甲盖大类描述保留；「垫纸吸油」类不被误改 |
| 甜品烘焙（戚风/曲奇/慕斯） | I 批：tools 完整（打发器/模具/裱花袋…），有替代给替代+代价，无替代标 null+原因；难度/时间合理 |
| 无口播纯字幕视频（可选，找得到就测） | VL 读屏兜底出有效菜谱 |

每条另附通用检查：steps 的 source_time 不超片长、confidence/风险标注合理、画面配图有图且不乱配、营养估算可触发、导出 Markdown 完整。

- **代码 bug**（崩溃/字段丢失/清洗漏洞/图片错位）：修复 + stub 测试 + 单独 commit
