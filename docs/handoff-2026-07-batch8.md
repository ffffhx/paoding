## H1：LLM 输出语言参数（小，先做）

`PAODING_OUTPUT_LANG`（默认 `zh`，可选 `en`）：structureRecipe / explainSteps / 营养估算 / 技法归纳 / AI 助手端点的 system prompt 末尾追加输出语言约束（zh 时**不追加任何东西**，保证现状零变化）。`.env.example` + README 说明。测试：用 E1 的 LLM stub 断言 en 时 prompt 含语言指令、zh 时与旧 prompt 完全一致。独立 commit。

## H2：英文 README（小）

`README.en.md`：精简翻译（是什么/为什么不一样/功能表/快速开始三方案/竞品对比表），两个 README 顶部互链语言徽章。不逐字翻译"自动部署"一节（那是作者私人运维笔记，英文版一句话带过）。独立 commit。

## H3：前端 i18n 基建 + 英文包（大头，按块分批提交）

1. 基建先行（独立 commit）：`app/i18n.js` —— `t(key, params?)` 函数 + `zh` 字典对象 + `en` 字典对象；语言设置存 userdata（设置页加"语言 / Language"选项，默认 zh）；`index.html` 引入。约定：**zh 字典的值就是现有中文原文**，key 用语义化短名。
2. 然后按界面分块抽取，**每块一个 commit**，建议顺序：设置页 → 首页/列表 → 菜谱详情 → 跟做模式 → 购物清单/计划 → 技法页 → 各类 toast/错误提示。没抽到的字符串保持中文硬编码——功能不受影响，这是安全网。
3. en 字典随每块同步补翻译；缺 key 时 `t()` 回退 zh 再回退 key 本身（写测试）。
4. 现有前端测试断言的是中文文案：默认 zh 下应全部照常通过（如需将断言改为经 t() 的取值，保持语义不变）。补 en 抽查测试（切 en 后关键 UI 字符串是英文）。
5. 日期/数字格式暂不做 locale 化（超范围，README 注明）。
