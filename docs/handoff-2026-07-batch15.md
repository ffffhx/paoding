## O1：甜品工具清单完整性兜底（戚风视频漏了模具/筛网/烤架/盆）

prompt 已要求完整但本地 14B 执行不全。加**规则兜底**（零额外 LLM 调用）：结构化后，甜品/烘焙类菜谱扫描步骤文本，命中动作词但 tools 缺对应工具时按推断补全（inferred: true，purpose 从动作生成）：
- 打发/打蛋 → 打蛋器；模具/脱模/倒入模 → 模具；过筛 → 筛网；烤箱/烘烤 → 烤箱；水浴 → 深烤盘；裱花 → 裱花袋；隔水/融化 → 耐热盆。
词表数据化、纯函数、配测试（含"非甜品不触发"回归）。

## O2：source_time 覆盖率（后半段步骤缺失导致无配图）

1. prompt 强化：转写带时间标记时，要求**每一步**都给 source_time（现在只说"额外输出"）；仍缺失的如实保留（诚实原则禁止外推，现有硬校验不动）。
2. 配图阶段：对缺 source_time 的步骤记录统计（日志/报告字段"N/M 步有时间戳"），便于观测改进效果。
3. stub 测试：prompt 文本断言 + 统计字段。

## O3：食材图阶段性能（真实视频中该阶段耗时数分钟）

排查 `vision.mjs` 食材特写路径：是否逐食材串行多次 VL 调用。优化：受控并发（2–3 路，尊重 AbortSignal）+ 相同帧复用 + 已失败食材不重试。保持行为与产物格式不变，stub 测试覆盖并发路径不丢结果、失败单个不毁整批。

## O4：K1 配方卡正向路径复验（上轮饮品视频没有真配方卡）

用 yt-dlp 搜索找一条**真的带画面配料表/配方卡**的中文短视频（关键词：配方 教程 字幕 / 烘焙配方 / 奶茶配方，优先 1–5 分钟；确认元数据后先低成本抽查几帧确认有文字卡再跑全链路）。跑通后核对：视觉转录含【画面配方卡】、用量入菜谱且 note 留痕出处。把结果补进 docs/verification-2026-07.md（K1 正向 ✅/仍 ⚠️ 如实写）并 commit。找不到合适视频就换关键词最多尝试 4 条，仍找不到如实记录。
